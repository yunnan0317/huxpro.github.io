# 一. 用大纲组织内容

orgmode使用大纲组织内容, 支持折叠文档, 针对大纲的编辑, 同时适应在文档中快速定位的需要, 在组织大型文档时尤为得力.

## 1.1 定义标题

orgmode使用星号(\*)进行定义, 用1到10个星号(\*)表示1到10级标题.

注意:
1. \*要位于每行的行首
2. \*之后要有一个空格, 然后再输入标题.

``` org
* org-mode
** 大纲
正在编写大纲
** 轻量级标记语言
* 可以导出其他格式
支持html, pdf等格式
```

## 1.2 大纲相关的快捷键

### 1.2.1 折叠大纲

快捷键|命令|说明
-|-|-
Shift-TAB|org-shifttab|循环切换整个文档的大纲状态
TAB|org-cycle|循环切换当前大纲的状态

### 1.2.2 在大纲之间移动

快捷键|命令|说明
-|-|-
C-c C-p/n||上/下一标题
C-c C-f/b||上/下一同级标题
C-c C-u||上一级标题
C-c C-j||大纲浏览状态


### 1.2.3 基于大纲的编辑

快捷键|命令|说明
-|-|-
M-Enter||插入同级标题
M-Shift-Enter||插入同级TODO标题
M-Left/Right||将当前标题升/降级
M-Shift-Left/Right||将子树升/降级
M-Shift-Up/Down||将子树上/下移
C-c *||将本行设为标题/正文
C-c C-w||将子树或区域移到另一标题处(跨缓冲区)
C-x n s/w||只显示当前子树/返回
C-c C-x b||在新缓冲区现实当前分支(类C-x n s)
C-c /||只列出包含搜索结果的大纲,并高亮,支持多种搜索方式
C-c C-c||取消高亮

## 1.3 大纲的显示方式


默认的大纲现实没有缩进,可以用`M-x org-indent-mode`切换到另一种显示方式.
若想让某个文件默认使用这种方式打开, 可在文件头部假如:

``` org
#+STARTUP: indent
```


若希望所有org文件都使用这种方式打开, 在 .emacs 中配置:


``` emacs-lisp
(setq org-startup-indented t)
```

# 二. 超链接和图文混排

org支持多种超链接, 包括图文混排.

## 2.1 外部链接

### 2.1.1 自动连接

对于符合链接规则的内容, org-mode会自动将其视为链接(包括文件, 网页, 邮箱, 新闻组, BBDB数据库, IRC会话和记录等). 下面链接会自动视为链接.

``` org
http://yunnan0317.github.com
file:/home/dominik/images/jupiter.jpg
/home/dominik/images/jupiter.jpg
file:paper/last.pdf

```

对于文件链接, 可以用 :: 后面增加定位符的方式链接到文件的特定位置. 定位符可以是行号或搜索选项. 如:

``` org
file:~/code/main.c::255     进入到目标文件255行
file:~/xx.rog::My Targe     找到目标'<<My Target>>;'
file:~/xx.org/::#my-custom-id   查找自定义 id 的项
```


### 2.1.2 指定现实链接

除了自动链接, 可以使用指定链接

``` org
[[link][description]]
[[link]]
```

### 2.1.3 快捷键

快捷键|命令|说明
-|-|-
C-c C-l|org-insert-link|创建或修改链接,可以引用已保存的链接
C-c l||保存链接
C-c C-o|org-open-at-point|打开链接
C-c %||记录内部链接地址
C-c &||跳转到已记录的内部链接


## 2.2 内部链接

建立内部链接:

``` org
定义锚点 #<<my-anchor>>
[[my-anchor][内部链接]
```


脚注可以看作是一种特殊的内部链接, 但要求有"fn: "前缀:

``` org
添加脚注链接 [[fn: footpring][脚注1]]
定义脚注 [fn: footpring1]
```

# 三. 轻量级标记语言

org与markdown一样, 也是一种轻量级标记语言.

## 3.1 字体

``` org
/斜体/
+删除线+
_下划线_
下标: H_2O
上标: E=mc^2
等宽字: =git= 或者 ~git~
```



## 3.2 表格

### 3.2.1 创建表格


使用\|创建.

快捷键|命令|说明
--|--|--
C-c \|||创建或转换成表格

### 3.2.2 调整和区域移动

快捷键|命令|说明
--|--|--
C-c C-c||调整表格, 不移动光标
TAB||移动到下一区域, 必要时创建新行
Shift-TAB||移动到上一区域
RET||移动到下一行, 必要时新建一行

### 3.2.3 编辑行和列

快捷键|命令|说明
--|--|--
M-Left/Right||移动列
M-Up/Down||移动行
M-Shift-Left/Right||删除/插入列
M-Shift-Up/Down||删除/插入
C-c -||添加水平分隔线
C-c Enter||添加水平分隔线并跳到下一行
C-c ^||根据当前排列, 可以选择排序方式




## 3.3 段落

单个回车换行的文本, 认为属于一个段落. 在导出时会转化为不换行的同一段. 新段要留出一个空行.

## 3.4 列表


* 无序列表以"-", "+"或者"*"开头.

* 有序列表以"1."或者"1)"开头.

* 描述列用"::"将项和描述分开.

* 有序列表和无序列表都以缩进表示层级. 只要对齐缩进, 不管是换行还是分块都认为是出于当前列表项.

``` org
Important Actors in this film are:
- Elijah Wood :: He plays Frodo
- Swan Austin :: He plays Sam,Frodo's frend.
```

显示为

``` org
Important actor in this film are:
Elijah Wood
He plays Frodo
Sean Austin
He plays Sam, Frodo's friend.
```

### 3.4.1 列表操作快捷键

快捷键|命令|说明
--|--|--
TAB||折叠列表项
M-Enter||插入项
M-Shift-Enter||插入带复选框的项
M-Shift-Up/Down||移动列表项
M-Left/Right||升/降级列表项, 不包括子项
M-Shift-Left/Right||升/降级列表项, 包括子项
C-c C-c||改变复选框状态
C-c -||更换列表标记(循环切换)

# 四. 标签

通常使用tag+category来管理信息.

## 4.1 标记tag

org-mode中, 可以对标题增加tag标记. 格式:

``` org
标题一     :tag1:tag2:tag3:

```

子标题会继承父标题的tag. 若希望文档中所有的标题都具有某些标签, 只要定义文档元数据:

``` org
#+FILETAGS: :Peter:Boss:Secret"
```

正文部分可以使用 C-c C-q 或直接在标题上用 C-c C-c 创建预定义的标签.

### 4.2 预定义tag

``` org
    #+TAGS: { 桌面(d) 服务器(s) } 编辑器(e) 浏览器(f) 多媒体(m) 压缩(z)
```

花括号表示标签组, 只能选择一个

### 4.3 按tag搜索

快捷键|命令|描述列用
--|--|--
C-c \||按tag搜索标签
C-c //m||搜索并按树状结构显示
C-c a m||按标签搜索多个文件

可以使用`+`, `-`, `|`和`&`加入逻辑条件.





# 五. 导出和发布

可以导出多种格式(包括txt, HTML, PDF, iCalendar等).

## 5.1 准备工作

### 5.1.1 文档元数据


``` org
#+TITLE:      文档标题(默认是缓冲区名字)
#+AUTHOR:     作者(默认是 user-full-name)
#+DATA:       日期
#+EMASIL:     电子邮箱地址(默认是 user-mail-address)
#+DESCRIPTION:页面描述
#+KEYWORDS:   页面的关键字
#+LANGUAGE:   HTML的语言(默认为英语 org-export-default-language)
#+TEXT:
#+OPTIONS: 一些设置
#+BIND:
#+LINK_UP     UP:链接
#+LINK_HOME   Home:接链
#+LATEX_HEADER
#+EXPORT_SELECT_TAGS
#+EXPORT_EXCLUDE_TAGS
#+XSLT:

```



其中#+OPTIONS是复合选项, 包括:


``` org
H: 输出的标题层数
num: 章节(标题)是否自动编号
toc: 是否生成索引, 索引到层数
\n: 保留\n标识
@: 保留HTML标签
::
|: 表格
^: TeX-like上下标
-: 保留连字符
f: 注脚
todo: TODO-list
tasks: TODO-list的item
pri: priority cookies
tags:
<: data/time stamps
    *: emphasized text (bold, italic, underline)
TeX: TeX macros
LaTeX: fragments
skip: skipping the text before
author:
email:
creator: 是否生成"creat by emacs..."提示
timestamp:
d: drawers
```
### 5.1.2 内容元数据

通常在行首以"#+"开头, 可以有多种用途.

* 分行区块

默认内容不换行, 需要留出空行才能换行, 分行区块可以实现普通换行

``` org
#+BEGIN_VERSE
    Great clouds overhand
    Tiny black birds rise and fall
    Snow conver Emacs
        -- AlexSchroeder
#+END_VERSE
```

* 缩进区块

通常用于引用, 与默认格式相比左右都会留出缩进.

``` org
#+BEGIN_QUOTE
这是一个缩进区块
#+END_QUOTE
```

* 居中区块

``` org
#+BEGIN_CENTER
这是一个居中区块
#+END_CENTER
```

* 代码区块

``` org
#+BEGIN_SRC ruby
    require 'redcarpet'
    md = Redcarpet.new("Hello, world.")
    puts md.to_html
#+END_SRC
```

* 举例

单行的例子以`:`开头

``` org
#+BEGIN_EXAMPLE
这是一个多行的例子
#+END_EXAMPLE
```

* 注释

以`#`开头的行被看做注释, 不会被导出

``` org
#+BEGIN_COMMENT
这是一个注释块
#+END_COMMENT
```

* Label和Caption

``` org
#+CAPTION: This is the caption for the next table (or link)
#+LABEL: tbl:table1
```

在需要的地方可以通过`ref{table1`引用

### 5.1.3 嵌入html

``` org
#+BEGIN_HTML

{
    return 0;
}
</pre>
</div>
#+END_HTML
```

### 5.1.4 包含文件

``` org
#+INCLUDE: "~/.emacs" src emacs-lisp
```

``` org
\begin{equation}
x=\sqrt{b}
\end{equation}
```
